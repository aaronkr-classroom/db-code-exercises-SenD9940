# chp3-실습과제.md

## I. 데이터 요구 분석 명세서

### 1. 여행 상품 관리
 - 새로운 여행상품을 계속 개발하고 있다.
 - 계절별로 판매하는 여행상품 수에 큰 차이가 있다.
 - 여행 상품 정보는 여행 번호와 출발지, 도착지, 프로그램 정보, 시작 일시, 종료 일시, 최소 출발 인원, 최대 인원 여행 경비, 예약금 정보를 관리한다

### 2. 예약 관리
 - 고객이 여행 상품을 예약한 날짜와 예약금 납부 여부를 관리한다.
 - 고객이 여행상품을 예약한 후 24시간 안에 예약금을 결제하지 않으면 예약을 취소한다
 - 여행 상품의 출발이 취소된 경우 1주일 안에 고객에게 예약금을 반환해야한다

### 3. 고객 관리
 - 고객 등급은 '일반', '우수', '최우수'로 구분한다.
 - 여생 상품에 6회 이상 참여한 고객은 최우수, 3~5회 참여한 고객은 우수, 나머지 고객은 일반 등급으로 구분한다
 - 고객 정보는 id와 이름 주민번호 휴대폰 번호 주소 그리고 등급을 관리한다

### 4. 직원 관리
 - 현재 약 30명의 직원이 근무하고 있다.
 - 직원별 담당 업무다 다르다.
 - 여행사 직원은 사번과 이름 주민등록번호, 휴대폰 번호, 급여, 담당 업무 입사일 정보를 관리한다


## II. 트랜잭션 요구 분석 명세서

### 1. 여행 상품 관리
 - 직원은 자신이 담당하는 여행 상품을 검색할 수 있다.
 - 새로운 여행상품이 개발되면 여행 상품을 등록할 수 있다
 - 고객은 예약을 위해 여행 상품 정보를 조회할 수 있다

### 2. 예약 관리
 - 고객이 여행 상품의 예약 일자 정보를 삽입힌다.
 - 고객이 여행 상품을 예약한 날짜와 예약금 납부 여부를 조회할 수 있다
 - 고객이 24시간 안에 입금을 하지 않아 예약이 취소될 경우 예약정보를 삭제할 수 있다

### 3. 고객 관리
 - 탈퇴한 고객 정보를 삭제한다.
 - 고객 등급을 여행 상품 참여도에 따라 수정할 수 있다
 - 직원은 예약자인 고객을 관리하기 위해 고객 정보를 조회할 수 있다

### 4. 직원 관리
 - 각 직원은 자신의 퓨대폰 정보를 변경할 수 있다.
 - 직원이 입사할 경우 직원 정보를 등록할 수 있다
 - 여행상품이 만들어진 경우 해당 여행상품에 담당 직원을 배정할 수 있다
